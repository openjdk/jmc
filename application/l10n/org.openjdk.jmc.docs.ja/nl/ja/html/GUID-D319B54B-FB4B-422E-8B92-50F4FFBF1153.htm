5 JMXコンソールについて

JMXコンソールで、Java Virtual Machine (JVM)の実行中のインスタンスやそのインスタンスで実行するアプリケーションをモニターおよび制御できます。

JMXコンソールは、メモリーとCPUの使用状況などのJavaアプリケーションの特性、およびガベージ・コレクション、メソッドの最適化などのJVMアクティビティに関するリアルタイムの情報を提供します。この情報は、開発環境において、アプリケーションのライフ・サイクルのどの時点で最もメモリーを消費するか見つけ出すことや、デプロイされた環境において、アプリケーション・サーバーのシステム・ヘルスをモニターすることなど、どちらの環境でも使用できます。

JMXコンソールはマルチタブを備えたインタフェースを提供します。各タブで、JVMやそのJVMで実行中のアプリケーションの一部分をモニターおよび制御できます。使用可能なタブのセットはインストール済のプラグインによって異なります。詳細は、JMXコンソールの使用を参照してください。

Javaプラットフォームは、アプリケーションのパフォーマンス、リソース消費およびロギング設定のモニタリングと管理を行うための、技術的に優れた設計を備えています。JDK Mission Controlは、JVMおよびJavaアプリケーションのリソースのモニタリングと管理のためにJava Management Extensions (JMX)インタフェースを使用します。

Simple Network Management Protocol (SNMP)や他の管理標準と同じく、JMXは広く知られた仕様であり、一般的に使用されているモニタリング製品の多くのベンダーがJMXをサポートしています。JMXではJava Runtime Environment (JRE)およびアプリケーションをモニターする標準的な方法が提供されます。JMX Remote Method Invocation (JMXRMI) APIを使用すると、そのモニタリングにリモートでアクセスできます。ツールは、管理対象サーバーに登録されているJMX Managed Bean (MBean)インタフェースからアクセスできます。アプリケーションが、独自のMBeanを作成して管理対象サーバーに登録することもできます(管理対象サーバーは、リモート・アクセスのためのシングル・ポイントとして使用できます)。JMX準拠クライアント(JMXコンソールなど)は、プラットフォーム管理サーバーに接続でき、JMXテクノロジを使用してJavaアプリケーションとJVMの両方を管理できます。JMX標準の詳細は、「Java Management Extensions API」を参照してください。

JMXコンソールを実行してJVMをモニターするためにかかる余分なコストはわずかであり、考慮しなくてもかまいません。それによって、アプリケーションのモニタリングやプロファイリングを低コストで実行できます。

注:

JMXコンソールをJVMと同じコンピュータで実行すると、JVMで実行されているアプリケーションが本来使用できるリソースをJMXコンソールGUIが消費します。JMXコンソールを使用せずにアプリケーションが実行されることを想定していた場合、これによって誤った分析が導き出される可能性があります。
