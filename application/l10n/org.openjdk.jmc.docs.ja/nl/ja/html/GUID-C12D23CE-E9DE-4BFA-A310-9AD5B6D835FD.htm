「ガベージ・コレクション」ページの使用

「ガベージ・コレクション」ページには、GCアクティビティに関連する情報が含まれます。

このページでは、ガベージ・コレクションおよびメモリー割当てに関する情報が表とグラフに表示されます。

「ガベージ・コレクション」ページの使用: 選択内容と側面

必要に応じて、ページの一番上にあるドロップダウン・メニューを使用して、選択内容と側面を指定します。他のページのデータを選択内容ドロップダウン・メニューに保存するには、チャートの領域をドラッグして(または表からデータを選択して)、コンテキスト・メニューから「選択内容の格納」または「格納して、フォーカスされた選択内容として設定」を選択します。

次のコントロールで追加の動作を指定します。

  • 同時の表示: 選択した側面に並行するイベントをすべて表示します。

  • 交差: 元のイベントと交差するすべてのイベントまたは同じ期間に含まれるイベントのみを表示します。

  • すべてのスレッド: すべてのスレッドのイベントまたは元のイベントに関連するスレッド内のイベントのみ表示します。

  • 時間範囲: ページ上の関連場所の時間範囲にズームするか、時間範囲のズームをクリアします。

「ガベージ・コレクション」ページの使用: 表とグラフ

表の列は、次のようにラベル付けされています。

  • GC ID: ガベージ・コレクションID。

  • 原因: このガベージ・コレクションのトリガーの理由。

  • コレクタ名: ガベージ・コレクタの名前。

  • 最長の一時休止: ガベージ・コレクション中の最長の個別一時休止。

  • 一時休止合計: ガベージ・コレクション中にJavaの実行が一時休止された合計期間。

  • 最終参照: 最終参照の数。

  • 弱い参照: 弱い参照の数。

  • ソフト参照: ソフト参照の数。

  • ファントム参照: ファントム参照の数。

グラフに表示する次の任意の項目を選択できます。
  • 使用されているヒープ: ヒープ上のオブジェクトによる割当て済メモリー。

  • ヒープ領域: コミットされたサイズ: 仮想領域のコミット済メモリーのサイズ。

  • ヒープ領域: 予約されたサイズ: 仮想領域の予約済メモリーのサイズ。

  • メタスペース: 使用済: メタスペース内のオブジェクトによる割当て済メモリー。

  • メタスペース: コミット済: メタスペースに対してコミットされたメモリー。

  • メタスペース: 予約済: メタスペースに対して予約されたメモリー。

  • 最長の一時休止: 最長GC一時休止の期間。

  • 一時休止合計: すべてのGC一時休止の合計期間。

  • 一時休止フェーズ: ガベージ・コレクションの一時休止のフェーズ。

「ガベージ・コレクション」ページの使用: ルールの構成

「構成の編集」アイコンをクリックすると、このページに関連付けられたルールの構成属性を設定できます。

次のオプションを使用できます。

GC一時休止ピーク期間

  • GC一時休止警告制限: 警告のトリガーに必要とされるGC一時休止時間。

GCロッカーによって引き起こされたGC

  • GCロッカー率制限: GCロッカーが原因のガベージ・コレクションの数と合計ガベージ・コレクション間の警告制限率。

ヒープ検査によって引き起こされたGC

  • ヒープ検査GC制限: ヒープ検査ガベージ・コレクションの数と合計ガベージ・コレクション間の警告制限率。

System.gc()によって引き起こされたGC

  • System.gc()率制限: System.gc()と合計ガベージ・コレクション間の比率の警告制限。
