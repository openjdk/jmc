更改 JVM

虽然 JDK Mission Control Client 可与许多不同的 JVM 一起工作，但是在 Eclipse 平台上运行 JDK Mission Control Client 时，强烈建议您使用 Oracle JDK/JVM 作为您的 JVM。确保使用 Oracle HotSpot JDK 7 Update 14 或更高版本。建议不要在 JRE 上运行，这将禁用本地 JVM 的自动检测。

要更改运行 Eclipse 的 JVM，请执行以下操作：

  1. 转到您的文件系统浏览器 (例如，Windows 资源管理器)。

  2. 定位您的 Eclipse 安装文件夹 (例如，C:\Program Files\Eclipse)，然后使用“记事本”之外的文件编辑器打开文件 eclipse.ini。该文件看起来类似于以下示例：


    -showsplash
    org.eclipse.platform
    --launcher.XXMaxPermSize
    256M
    -vmargs
    -Dosgi.requiredJavaVersion=1.5
    -Xms40m
    -Xmx512m

  3. 对 eclipse.ini 文件进行以下更改：

    • 删除与 JVM 之外的其他 JVM 相关的所有标记 (例如 --launcher.XXMaxPermSize 256M)。

    • 在第三行上 (org.eclipse.platform 之后)，添加以下内容：

      -vm <Full path JAVA_HOME/jre/bin/ folder>

      在 Windows 上，完整路径可能如下所示：

      C:\Program Files\Java\jdk1.7.0\jre\bin\

      或者，在 Linux 和 Solaris 上，该路径可能如下所示：

      $HOME/jdk1.7.0/jre/bin/

    • 根据您的特定 JVM 实施和其上运行的应用程序，可以设置任何有效的 JVM 命令行选项。

      有关可用命令行选项的详细信息，请参阅 Java 命令行参考，网址为：
      https://docs.oracle.com/javase/7/docs/technotes/tools/solaris/java.html (Solaris、Linux)
      https://docs.oracle.com/javase/7/docs/technotes/tools/windows/java.html (Windows)

  4. 完成对 eclipse.ini 文件的必要更改后，保存并关闭该文件。以下示例显示为了使 JVM 在 Windows 实施中成为活动 JVM 而更新的 eclipse.ini 文件。


    -showsplash
    org.eclipse.platform
    -vm
    C:\Program Files\Java\jdk1.7.0\bin\javaw.exe
    -vmargs
    -Dosgi.requiredJavaVersion=1.5
    -Xms256m
    -Xmx512m
