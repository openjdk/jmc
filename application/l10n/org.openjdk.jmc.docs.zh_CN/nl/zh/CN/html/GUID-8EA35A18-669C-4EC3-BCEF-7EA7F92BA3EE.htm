使用“Java 应用程序”页

Java 应用程序页高度概述了执行此记录时运行的应用程序。

此页包含以表和图形形式显示的每个线程的 I/O 时间、阻止时间、类加载时间和内存分配估计值。

单击水平布局或垂直布局图标可以设置布局方向。

使用“Java 应用程序”页：选择和方面

使用页顶部的下拉菜单选择选项和方面（如果需要）。通过拖动图表区域（或选择表中的数据）并从上下文菜单中选择存储选择或存储选择并设置为焦点选择，可以将其他页中的数据保存到选择下拉菜单。

下面的控件提供其他行为：

  • 显示并发：显示对于所选方面并发的所有事件。

  • 交集：显示与原始事件有交集的所有事件，或者仅显示包含在同一时间范围内的事件。

  • 所有线程：显示所有线程中的事件，或者仅显示与原始事件相关的线程中的事件。

  • 时间范围：缩放到页上的相关位置的时间范围，或者清除时间范围缩放。

使用“Java 应用程序”页：表和图形

表中各列的标签如下：
  • 线程：线程的名称。

  • 概要分析样本：此线程的方法概要分析样本数。

  • I/O 总时间：线程忙于 I/O 的估计总时间。实际总时间可能更长。

  • 总阻塞时间：监视器阻塞线程的估计总时间。实际持续时间可能更长。

  • 类加载时间：线程加载类所用的估计总时间。实际持续时间可能更长。

  • 总分配量：估计的总分配量。此值不是确切值。

  • 可引发异常错误：已创建的可引发异常错误数。

您可以选择以下任一项以便在图形上显示：

  • 中断：JVM 中断用户代码执行（例如由于垃圾收集）的时段。

  • 计算机总数：计算机上的所有进程使用的 CPU。

  • JVM + 应用程序：JVM 进程使用的 CPU 总量。

  • 概要分析：概要分析样本数。颜色表示顶级堆栈跟踪帧的分布（红色 = 少数几个方法具有最多样本，黄色 = 样本均匀分布）。

  • 总分配量：估计的总分配量。此值不是确切值。

  • 可引发异常错误：已创建的可引发异常错误数。

  • 线程活动：线程活动通道。使用上下文菜单配置单独的通道。

使用“Java 应用程序”页：配置规则

通过单击编辑配置图标，可以设置与此页关联的规则的配置属性。

可以使用以下选项：

高 JVM CPU 负载

  • JVM CPU 占用率信息限制：触发信息通知时 JVM 需使用的 CPU 量。

  • 最短 CPU 负载时段：触发警告所需的最短 CPU 负载时段。

  • 缺少样本限制：触发警告所需的最少缺少样本数。

热点方法

并行线程

  • 采样线程比率限制：触发警告所需的采样线程数与硬件线程数之间的比率。
