使用“垃圾收集”页

垃圾收集页包含与 GC 活动相关的信息。

此页以表和图形的形式显示垃圾收集和内存分配的相关信息。

使用“垃圾收集”页：选择和方面

使用页顶部的下拉菜单选择选项和方面（如果需要）。通过拖动图表区域（或选择表中的数据）并从上下文菜单中选择存储选择或存储选择并设置为焦点选择，可以将其他页中的数据保存到选择下拉菜单。

下面的控件提供其他行为：

  • 显示并发：显示对于所选方面并发的所有事件。

  • 交集：显示与原始事件有交集的所有事件，或者仅显示包含在同一时间范围内的事件。

  • 所有线程：显示所有线程中的事件，或者仅显示与原始事件相关的线程中的事件。

  • 时间范围：缩放到页上的相关位置的时间范围，或者清除时间范围缩放。

使用“垃圾收集”页：表和图形

表中各列的标签如下：

  • GC ID：垃圾收集 ID。

  • 原因：触发此垃圾收集的原因。

  • 收集器名称：垃圾收集器的名称。

  • 最长的暂停：垃圾收集期间最长的单次暂停。

  • 暂停总和：垃圾收集期间 Java 执行暂停的总持续时间。

  • 最终引用：最终引用的数目。

  • 弱引用：弱引用的数目。

  • 软引用：软引用的数目。

  • 幻像引用：幻像引用的数目。

您可以选择以下任一项以便在图形上显示：
  • 已用堆：堆上按对象分配的内存。

  • 堆空间：已提交大小：虚拟空间的已提交内存大小。

  • 堆空间：保留大小：虚拟空间的保留内存大小。

  • 元空间：已用：元空间中按对象分配的内存。

  • 元空间：已提交：为元空间提交的内存。

  • 元空间：已保留：为元空间保留的内存。

  • 最长的暂停：最长 GC 暂停的持续时间。

  • 暂停总和：所有 GC 暂停的总持续时间。

  • 暂停阶段：垃圾收集暂停的阶段。

使用“垃圾收集”页：配置规则

通过单击编辑配置图标，可以设置与此页关联的规则的配置属性。

以下选项可供使用：

GC 暂停峰值持续时间

  • GC 暂停警告限制：触发警告所需的 GC 暂停时间。

GC 锁定器导致的 GC

  • GC 锁定器比率限制：GC 锁定器导致的垃圾收集数与垃圾收集总数之间的警告限制比率。

堆检测导致的 GC

  • 堆检测 GC 限制：堆检测垃圾收集数与垃圾收集总数之间的警告限制比率。

System.gc() 导致的 GC

  • System.gc() 比率限制：System.gc() 与垃圾收集总数之间比率的警告限制。
