启动 JDK Mission Control Client

JMC 客户机可执行文件位于 Java SE 开发工具包 (JDK) 安装路径 (JAVA_HOME) 的 bin 目录中。

如果 JAVA_HOME/bin 目录位于 PATH 环境变量中，则可以通过在命令行提示 (shell) 中输入 jmc 来启动 JMC 客户机。否则，必须指定 JMC 可执行文件的完整路径：
  • JAVA_HOME\bin\jmc.exe (Windows)
  • JAVA_HOME/bin/jmc（Linux 和 OS X）

注：

在 Windows 上，将向开始菜单中的所有程序下的 Java 开发工具包文件夹中添加 JMC 客户机可执行文件的快捷方式。

注：

在 OS X 上，如果找不到 JMC 可执行文件或者无法在 Spotlight 中定位 JMC，则可以从命令行使用 /Library/Java/JavaVirtualMachines/jdk<version>.jdk/Contents/Home/bin/jmc 来启动 JMC。此外，使用 Finder 浏览到 /Library/Java/JavaVirtualMachines/jdk<version>.jdk/Contents/Home/lib/missioncontrol 并双击 JDK Mission Control 应用程序。

注：

在 OS X 上，当启动 JMC 可执行文件时，将打开“终端”窗口。关闭 JMC 客户机后，“终端”窗口将保留，其中显示文本 [Process completed]。要避免此问题，请打开“终端”的首选项，然后打开设置选项卡，选择要使用的方案，然后选择 Shell 选项卡，将当 Shell 退出时选项更改为当 Shell 完全退出时关闭。

将 JVM 选项传递到 JMC 启动程序

JMC 是 Java 应用程序，JMC 客户机可执行文件是此应用程序的启动程序。JMC 启动由在 jmc.ini 文件（该文件位于 JAVA_HOME/lib/missioncontrol 目录中）中指定的选项控制。jmc.ini 文件中的 -vmargs 选项的参数是传递到运行 JMC 应用程序的 JVM 的选项。可以指定这些选项以控制此 JVM 的运行方式。如果不需要修改 jmc.ini 文件，则可以在命令行上将 JVM 选项指定为 jmc 命令的 -vmargs 选项的参数。

要使用您自己的一组 JVM 选项启动 JMC 客户机（这些选项将覆盖在 jmc.ini 文件中指定的选项），请运行以下命令（用空格分隔多个参数）：

jmc -vmarg arguments

注：

如果为 jmc 命令指定了其他选项，则必须最后指定 -vmargs 选项。

要使用其他 JVM 选项启动 JMC 客户机（将这些选项附加到在 jmc.ini 文件中指定的选项），请运行以下命令（用空格分隔多个参数）：

jmc --launcher.appendVmargs -vmarg arguments

使用工作区目录

如果要将您的 JMC 客户机设置复制到另一台计算机或另一个用户，或为不同的应用程序使用不同的预定义设置，则可以在启动 JMC 客户机时添加 -data 命令行选项并定义工作区目录：

jmc -data workspace-directory
