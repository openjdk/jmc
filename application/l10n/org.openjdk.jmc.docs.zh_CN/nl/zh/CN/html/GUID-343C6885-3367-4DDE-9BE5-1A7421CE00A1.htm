使用“方法概要分析”页

方法概要分析页包含有关采样堆栈跟踪的信息。

此页以表的形式显示方法概要分析信息。

此方法概要分析不要求线程处于安全点即可对堆栈跟踪采样。但是，由于常常仅在安全点遍历堆栈，因此 Java HotSpot 虚拟机通常不会为非安全点的代码部分提供元数据。这意味着，除非您指定以下内容，否则这些样本将不会正确解析为正确的行号和 BCI：
-XX:+UnlockDiagnosticVMOptions -XX:+DebugNonSafepoints

通过使用 DebugNonSafepoints，编译器还将为不在安全点的代码部分生成所需的元数据。

使用“方法概要分析”页：选择和方面

使用页顶部的下拉菜单选择选项和方面（如果需要）。通过拖动图表区域（或选择表中的数据）并从上下文菜单中选择存储选择或存储选择并设置为焦点选择，可以将其他页中的数据保存到选择下拉菜单。

下面的控件提供其他行为：

  • 显示并发：显示对于所选方面并发的所有事件。

  • 交集：显示与原始事件有交集的所有事件，或者仅显示包含在同一时间范围内的事件。

  • 所有线程：显示所有线程中的事件，或者仅显示与原始事件相关的线程中的事件。

使用“方法概要分析”页：表

表中各列的标签如下：

  • 程序包：输入全限定程序包名。

  • 类：全限定类名。

  • 计数：对方法采样的次数。
