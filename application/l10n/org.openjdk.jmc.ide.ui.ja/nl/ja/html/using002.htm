JDK Mission ControlのEclipse IDEバージョンとRCPバージョンの相違

一般的に、EclipseバージョンのJDK Mission Controlクライアントの動作はRCPバージョンと同じです。Eclipseバージョンのすべてのコンポーネントは、対応するコンポーネントがRCPで実現するのと同じ機能とユーザー・インタフェースを提供します。インストール後にJDK Mission Controlに切り替えるには、Mission Controlのパースペクティブに移動します。

EclipseバージョンとRCPバージョンの最大の違いは、Jump-to-Source機能です。Jump-to-Source (アプリケーション・ソースへの移動を参照)は、モニタリングおよびプロファイリングのツールを開発環境に密接に結び付けることで実現しました。ソースがワークスペース・ソース・パス上にある場合は、ソースへのジャンプにより、Mission Controlに表示されているクラス、メソッドまたはフレームから対応するソースを開いて表示できます。
